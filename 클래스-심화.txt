클래스-심화

기초실습
< Partical 클래스와 협업 >
Partial Class나 파일 분할이 협업 시 왜 도움이 되는지 정리해 서술하고, 보고서를 제출하시오.
----------------------------------------------------------------------------------------------------------
여러 파일로 나누어 동일한 클래스를 관리함으로써 여러 명의 개발자가 하나의 프로젝트를 동시에 작업할 때 유용하다. 각 파일을 통해 담당 개발자가 맡은 부분에 대해서만 관리할 수 있기 때문에 가독성 또한 향상된다. 이 외에도 변경된 한 부분만 다시 컴파일 할 수 있기 때문에 전체 프로젝트를 다시 빌드하지 않아도 되어 더 빠른 빌드시간을 제공해주기도 한다.
즉, 코드의 구조를 더 모듈화 할 수 있고 다수의 개발자가 협업할 때 효율성을 높일 수 있다.


심화과제
< 깊은 복사 심화 >
1) 깊은 복사를 통해 생성된 객체가 가진 참조형 변수는 기존 객체의 변수와 메모리 공간상으로 같은 위치에 존재하는가?
: 깊은 복사를 통해 생성된 객체가 가진 참조형 변수는 기존 객체의 변수와 메모리는 서로 다른 메모리 공간에 있다.

2. 기존 플레이어의 변수와 새로 할당된 변수의 내용은 어떻게 다른가?
:  깊은 복사의 경우 다른 변수에 값만을 복사하기 때문에 기존 변수와 새로운 변수는 
  서로 다른 변수이다. 모습은 똑같아도 다른 친구이기 때문에 값을 변경해도 혼자만 변경됨

3. 새로 생성된 객체의 변수 내부 사항까지 복사하려면 어떻게 해야 하는가?
: ref키워드를 통해 새로 생성된 객체의 주소값을 가지고 와서 값을 복사해오거나 class와 같은 참조타입으로 객체를 선언하면 주소를 가져오는 얕은 복사를 사용한다.
